## Key Decisions Made

1. **Transaction ID is the key to reply bypass** — X requires `x-client-transaction-id` header on GraphQL requests. Without it, error 226. Generated by: fetch X homepage → extract verification key from meta tag → fetch ondemand.s JS → parse indices → SVG animation cubic bezier math → SHA256 → base64. Ported from Python (github.com/iSarabjitDhiman/XClientTransaction MIT) to native Rust.

2. **Chrome v130+ domain hash** — Chrome DB version >= 24 means 32-byte SHA256(domain) prepended to plaintext before AES encryption. The `meta` table stores version as TEXT not INTEGER; must parse as string then convert.

3. **X API minimum counts** — search/replies require `max_results >= 10` (API returns 400 otherwise). User tweets/mentions require >= 5. Followers/following accept >= 1. Do NOT set all to clamp(1, 100) — this was tried and broke search.

4. **Preflight scoring is for standalone posts only** — replies in threads get penalised for "weak hook", "no numbers" etc. which are irrelevant in reply context. Score/grade/warnings omitted from reply output.

5. **style.voice is LLM-interpreted, not Rust-enforced** — attempted Rust-side style parsing (em dash detection, filler words) but this hardcodes one user's preferences. Free text; LLM reads from agent-info and writes accordingly.

6. **GraphQL query hash rotates every 1-3 weeks** — currently `oB-5XsHNAbjvARJEc8CZFw`. Override via `keys.graphql_create_tweet_id` in config. The transaction ID ondemand.s file also rotates but is fetched live each time.

### Priority 1: Exit code consistency (Codex finding #1)
Several commands emit error JSON but exit 0 because they call `render_error()` then `return Ok(())`. An agent checking `$?` thinks it succeeded.

**Files to fix:**
- `src/commands/schedule.rs:292` — no scheduled posts → exits 0
- `src/commands/report.rs:16` — no data → exits 0
- `src/commands/suggest.rs:16` — no data → exits 0
- `src/commands/track.rs:27` — no data → exits 0
- `src/commands/bookmarks_cmd.rs:198` — unread with no data → exits 0
- `src/commands/engage_recommend.rs:167` — no targets → exits 0

**Fix pattern:** Change `render_error(...); return Ok(())` to `return Err(XmasterError::NotFound(...))` so the error propagates through main.rs and gets the right exit code.

### Priority 2: web-login JSON framing (Codex finding #2)
`config web-login` calls `set()` twice (which each emit JSON), then emits a third JSON object. Any parser expecting one JSON document gets broken.

**Fix:** Suppress JSON output from the intermediate `set()` calls. Only emit the final `WebLoginResult`.
- File: `src/commands/config_cmd.rs:396-406`

### Priority 3: update --check panic (Codex finding #3)
`xmaster update --check` panics with Tokio runtime drop error (exit 101). The `self_update` crate likely blocks inside async context.

**Fix:** Wrap in `tokio::task::spawn_blocking` or run the update check before entering the async runtime.
- File: `src/commands/update.rs`

### Priority 4: track status DB schema error (Codex finding #7)
`xmaster track status` fails with "Invalid column type Text at index: 2, name: posted_at". The SQLite DB was created by an older version without proper schema migration.

**Fix:** Add `ALTER TABLE` migration or recreate the table with correct types. Check `CREATE TABLE IF NOT EXISTS` in:
- `src/intel/tracker.rs:124`
- `src/intel/store.rs:88`

### Priority 5: Schema normalisation (Codex finding #6)

Pick one canonical schema and enforce it everywhere. Suggested canonical:

**Fix:** Add `tracing_subscriber::fmt::init()` or `EnvFilter` setup in main.rs.

## Gotchas & Warnings

- **Do NOT change clamp minimums on search/replies below 10** — X API returns 400. This was tried and reverted. Search needs `clamp(10, 100)`, timelines need `clamp(5, 100)`.
- **The transaction_id.rs module depends on X's homepage structure** — if X changes their meta tags, SVG animation IDs, or ondemand.s file format, it breaks. The regex patterns in `extract_meta_content`, `extract_anim_svg_paths`, and `extract_indices` are the fragile points.
- **Chrome cookie extraction is macOS-only** — uses Keychain via `security` CLI and macOS paths.
- **Boris's writing style is saved in config** — `style.voice` field. Don't overwrite.
- **The `engage recommend` command returned empty in testing** — it depends on having mentions/engagement history in the intel store. Works when there's data but gives no useful error when empty.
- **`self_update` crate has Tokio compatibility issues** — the `update --check` panic is because it does blocking I/O inside an async context. Use `spawn_blocking`.
- **The Codex audit ran actual commands** — `schedule list`, `track status`, `update --check` — and found real runtime failures. These are not theoretical.
